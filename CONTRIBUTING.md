# Contributing to Movie Recommendation System

Thank you for considering a contribution! This guide helps keep the repository clean and consistent.

## Forking

1. Fork the repository to your GitHub account.
2. Clone your fork locally.
3. Create a feature branch for your changes.

## Branch Naming

Use descriptive branch names, for example:

- `feature/streamlit-ui`
- `fix/data-loading`
- `chore/documentation`

## Pull Request Guidelines

- Keep changes focused and small.
- Write a clear title and description.
- Explain what the change does and why it is needed.
- Ensure all tests pass before opening a PR.

## Code Style

- Follow standard Python formatting (PEP 8).
- Use clear and descriptive names for variables and functions.
- Add comments and docstrings where appropriate.
- Keep the user interface clean and accessible.

## Testing

Run the test suite before submitting changes:

```bash
pytest
```

## Reporting Issues

If you find a bug or want to suggest a feature, please open an issue with a clear description and reproduction steps.
